# Test Driven Development

- **TDD (Test Driven Development)** — a software development practice where business requirements are converted into test cases before writing code.
- TDD cycle — **Red → Green → Refactor**.
  1. **Red** — write a test that fails (feature not yet implemented).
  2. **Green** — write just enough code to make the test pass.
  3. **Refactor** — clean up the code while keeping the test passing.
- In TDD.
  1. Write tests first.
  2. Write code to pass the tests.
- Why TDD.
  - Ensures business requirements are documented in tests.
  - If a requirement changes or a bug is found — update tests first, then adapt the code.
  - Ensures better code quality in the long run.
  - Helps catch bugs early — reduces defects by 40% to 90% compared to non-TDD teams.
