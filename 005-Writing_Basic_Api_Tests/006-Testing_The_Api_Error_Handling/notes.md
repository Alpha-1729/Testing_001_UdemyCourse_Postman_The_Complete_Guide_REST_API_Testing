# Testing The Api Error Handling

- **Happy Path Testing** — verifies that the basic functionality of the API works correctly.
  - Get all products.
  - Add item to cart.
  - Create an order.
- **Negative Testing** — focuses on how the API behaves in exceptional or edge cases.
  - Example: sending an invalid product ID, missing required fields, or unauthorized requests.
- Postman allows adding folders inside a collection to organize requests.
  - Any number of sub-folders can be added inside a collection.
